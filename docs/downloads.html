Datasets & downloads

Two datasets, normalised on mark_id: the marks (metadata, in git) and the episodes rows (per-unit, in object storage). The episode rows are published per mark — one gzipped CSV for each mark — so a mark's rows are its file.

The episodes dataset primary

Each unit's (state, action, reward) row: its context before the decision (running_value and the mark's covariates), what was done (assigned/treated), and the outcome that followed. Published per mark as a gzipped CSV; join its rows to the mark JSON on the mark id for the design and the full effect distribution. (RDD/DiD marks ship a cross-section row per unit; ITS marks ship a panel row per series × time bucket.)

4297 rows total across 5 marks · one gzipped CSV per mark
Columns: unit_id, unit_name, running_value, assigned, treated, outcome (+ each mark's covariates)
