Datasets & downloads

Two datasets, normalised on mark_id: the marks (metadata, in git) and the unified episodes dataset (the per-unit rows, in object storage). Everything is referenced by URL + SHA-256, so a download is verifiable on arrival.

The episodes dataset primary

Every mark's per-unit rows, unioned into one row-per-unit (state, action, reward) table. One Parquet; filter on mark_id for a single mark's rows. The same bytes load from Hugging Face as the episodes config.
